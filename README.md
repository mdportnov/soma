# Soma — Personal Health Dashboard

Self-hosted, **privacy-first / local-first** desktop app: one place for lab results, doctor
visits, diagnoses, medications and supplements — built for people without a permanent medical
record who move between countries and run regular blood-work cycles.

The key differentiator: **medication overlays on biomarker trend charts** — see "took X at dose Y →
biomarker Z shifted" on a single chart, plus a unified horizontal timeline of every health event.

## Stack

| Layer | Tech |
|---|---|
| Shell | [Tauri 2](https://tauri.app) (Rust) |
| Frontend | React 19 + TypeScript, Vite |
| UI | Tailwind CSS v4, shadcn-style components, lucide icons |
| Charts | Recharts (trend charts with reference/optimal bands) |
| Database | SQLite via `tauri-plugin-sql`, **Drizzle ORM** (schema + generated migrations) |
| AI | Multi-provider (Anthropic / OpenAI / Gemini / OpenRouter), user's own key, multimodal models only |
| Secrets | OS keychain via the Rust `keyring` crate |

## Development

Prerequisites: Node 22+, pnpm, Rust toolchain, plus the
[Tauri system dependencies](https://tauri.app/start/prerequisites/) for your OS
(on Linux: `webkit2gtk-4.1`, `libayatana-appindicator`, etc.).

```bash
pnpm install
pnpm tauri dev      # run the desktop app
pnpm tauri build    # production bundle
```

Useful scripts:

```bash
pnpm typecheck      # tsc --noEmit
pnpm build          # typecheck + vite build (frontend only)
pnpm db:generate    # regenerate Drizzle migrations after editing src/db/schema.ts
node scripts/gen-icons.mjs  # regenerate placeholder icons
```

## Architecture

```
src/
  db/
    schema.ts            # Drizzle schema — single source of truth (§3 of the spec)
    migrations/          # generated by drizzle-kit, applied in-app at startup
    client.ts            # Drizzle over tauri-plugin-sql (sqlite-proxy driver)
    migrate.ts           # startup migrator (tracks applied files in __migrations)
    seed-biomarkers.ts   # ~65 biomarkers with ranges + aliases (EN/RU) for the import mapper
    repos.ts             # typed data access + unified timeline selector
  ai/
    model-registry.json  # configurable model list (vision/pdf flags) — no hardcoded models in logic
    types.ts             # AIProvider interface: extractFromDocument / mapBiomarker / chat
    providers/           # one adapter per vendor; pipeline never touches SDKs
    pipeline/map.ts      # phase-2 deterministic mapping: normalize → exact/alias → fuzzy → narrow AI
    keystore.ts          # OS-keychain bridge (Rust commands)
  pages/                 # Dashboard, Timeline, Biomarkers(+detail), Labs(+new/detail/import),
                         # Medications, Visits(+detail), Diagnoses, Settings
  components/
    charts/TrendChart.tsx          # ref/optimal bands + medication overlay
    charts/HorizontalTimeline.tsx  # all events on one horizontal time scale
src-tauri/               # Rust shell: plugins (sql/dialog/fs/http/opener) + keychain commands
```

### AI import pipeline (two-phase + mandatory review)

1. **Extraction** — the document (PDF/photo, any language) goes to the selected multimodal model
   with a strict JSON-only prompt: `{raw_label, value, unit, ref_range_text, page}` — original
   labels verbatim, no interpretation, no dictionary mapping.
2. **Mapping** — deterministic, in code: label normalization → exact/alias match (incl.
   translations) → trigram/Levenshtein fuzzy match → only for leftovers, a narrow AI call that may
   *only pick from an explicit candidate list or return null*. Unit conflicts are converted via a
   per-analyte conversion table; unknown conversions are flagged for manual confirmation.
3. **Review (mandatory)** — confidence-colored table (green exact / yellow fuzzy / blue AI / red
   unmatched), editable mappings, duplicate detection, custom-biomarker creation. Only confirmed
   rows are written to `lab_result`, each keeping its `raw_label` for audit.

### Privacy model

- Everything lives in a local SQLite file; the app is fully offline-capable.
- AI is **off by default**; every AI surface shows a stub until a key is added in Settings.
- API keys are stored in the OS keychain — never in the DB or any config file.
- Documents are sent to the AI provider only on an explicit user action.
- Full JSON/CSV export — no lock-in. AI outputs carry a "not medical advice" disclaimer.
